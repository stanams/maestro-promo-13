## EPIC: Double authentification (2FA)

Login user => Demande de verification par sms => Indiquer mon code du sms => Autorisation pour garder en mémoire le nav.

US1 : 
  En tant qu'utilisateur je suis connecté sur mon compte, 
  et je veux recevoir un sms
  afin de sécuriser ma connexion.

**Critères d'acceptance** : 
* Etant donné que l'utilisateur renseigne son numéro de téléphonne
- le champ du formulaire vérifie qu'il commence par 06 ou 07 et est une suite de 10 chiffres.
* Etant donné que le numéro de téléphonne renseigné dans le formulaire est au bon format
- un code de validation est envoyé par sms l'utilisateur renseinge le code qu'il à reçu.
* Etant donné que l'utilisateur à bien reçu le sms avec le code de confirmation
- l'orsque que le champ du chiffre est rempli le curseur se déplace
- quand je saisie une lettre, le champ se vide, il a une animation qui vibre, et le telephonne vibre
- un système de sécurité est mis en place pour le brutforce au bout de 3 tentatives fausses bloquer le compte
* Etant donné que l'utilisateur à rempli dernier chiffre du code
- une requette est envoyée pour verifier que le code correspond bien

US2 : 
  En tant qu'utilisateur qui vient d'être loggé, 
  je veux  remplir avec le code du sms
  afin de pouvoir passer à l'étape suivante 

**Critères d'acceptance** : 

  - Etant donné que j'ai reçu un sms et que suis connecté quand je rentre le code dans le champ alors je suis redirigé auto. vers "Simplifier votre     connexion"
  - Etant donné que j'ai reçu un sms et que je suis non connecté
  - Etant donné que j'ai reçu un sms expiré et que je suis connecté
  - Etant donné que j'ai reçu un sms et que suis connecté  quand je saisis un chiffre dans le champ, je passe automatiquement au champ suivant pour le prochain chiffre
  - Etant donné que j'ai reçu un sms et que suis connecté quand je saisis une lettre ou un caractère special, je ne peux pas remplir le champ et je suis notifié par un une vibration.
  - Etant donné que j'ai saisie le dernier champ  et que je suis connecté , mon code est validé , je passe à l'étape suivante.
  
US3 : 
  En tant qu'utilisateur non loggé,
  je veux pouvoir rentrer le code à 5 chiffres reçu par sms
  afin de valider ma deuxième authentification.
  
**Critères d'acceptance** : champs pour intégrer les cinq chiffres à la main

US3 update :
En tant qu'utilisateur non loggé, 
je rentre le code à 5 chiffres reçu 
afin de valider ma deuxième authentification.

Cas1
Etant donné que je reçois un code sms, 
quand je le note correctement dans le champs qui s'affiche, 
alors je me logge.
Cas 2
Etant donné que je mets mon premier chiffre
quand j'inscris mon code, 
alors le curseur se déplace sur le champs suivant
Cas 3
Etant donné que je suis sur le dernier champs, 
quand j'ai rempli le formulaire du code, 
alors une requête est envoyée pour vérifier que le code est bon.

  
  US4
  En tant qu'utilisateur non loggé,
  je veux qu'un code me sois envoyé par email,
  afin de valider ma deuxième authentification
  
  **Critères d'acceptance** : bouton cliquer pour envoyer un mail
